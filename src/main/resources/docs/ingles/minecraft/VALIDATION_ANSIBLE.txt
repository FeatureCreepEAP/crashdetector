CrashDetector Ansible integration validation
============================================

Validation date: 2026-07-16
Target language level: Java 8

Completed checks
----------------

1. Compiled all Ansible integration classes, the modified CrashDetectorCli,
   and the existing container integration classes with `javac --release 8`.

2. Recompiled with `-Xlint:all`.
   Result: no source warnings; only JDK warnings that Java 8 source/target is
   obsolete in the installed modern compiler.

3. Tested ansible-runner selection with a controlled executable:
   - `--version` discovery
   - Runner command construction
   - private_data_dir creation
   - two per-host captured logs
   - delivery of two independent temporary inputs to MonitorDePID

4. Tested automatic fallback to ansible-playbook when Runner is unavailable.

5. Tested external playbook safety gate:
   - execution rejected without `--allow-external-playbook`
   - execution accepted with explicit consent
   - output delivered to MonitorDePID

6. Tested executor protections:
   - timeout destroys a slow process
   - stdout limit destroys an excessive-output process
   - stdin is closed and no interactive input is supplied

7. Parsed the generated built-in playbook with PyYAML.
   Result: valid YAML, including paths containing apostrophes.

8. Compiled the generated callback plugin with Python `py_compile`.

9. Simulated Ansible callback results:
   - host names are sanitised and hashed
   - metadata and stdout are written to a UTF-8 `.log`
   - over-limit output produces a `.too-large` marker

10. Tested help routing so `ansible logs --help` does not print a false error.

Not available in this environment
---------------------------------

The environment did not contain a real ansible-core or ansible-runner
installation, SSH-managed hosts, vault credentials, or a live inventory.
Therefore actual network authentication and execution against real managed
nodes were not tested. The command forms and private-data layout were checked
against current official Ansible and Ansible Runner documentation.
